# 4. 数据需求

本章描述“白马程序员内部”前端产品在运行过程中需要展示、采集、维护、统计和交换的数据。系统定位为面向机构内部人员的 Web 前端管理产品，主要服务于课程运营、学员管理、班级管理、讲师与教研管理、就业服务、内容资源维护、数据看板和系统权限管理等内部业务场景。数据需求应保证前端页面展示一致、表单录入规范、接口传输明确、报表统计可追溯，并为后续与后端服务、权限系统、消息系统和文件资源服务集成提供依据。

## 4.1 逻辑数据模型

“白马程序员内部”系统的核心数据实体包括：内部用户、角色、权限、学员、班级、课程、课程章节、教学资源、讲师、学习进度、作业、作业提交、考试、考试成绩、就业服务记录、企业、通知公告、操作日志、图片资源和统计报表。各实体之间的主要关系如下。

1. 一个内部用户可以拥有一个或多个角色；一个角色可以关联多项菜单权限、按钮权限和数据权限。
2. 一个学员可以归属一个或多个学习班级；一个班级对应一个课程方向，并由班主任、讲师或运营人员负责维护。
3. 一个课程包含多个课程章节；一个课程章节可以关联多个教学资源、作业或考试。
4. 一个学员对一个课程或章节产生一条或多条学习进度记录；学习进度用于驱动学习看板、学员跟进和预警提示。
5. 一个作业可以由多个学员提交；每条作业提交记录对应提交内容、附件、批改状态和评分结果。
6. 一次考试可以对应多个学员成绩；考试成绩与课程、班级、学员关联，用于统计通过率、平均分和风险学员。
7. 一个就业服务记录对应一个学员在某一阶段的求职跟进情况；就业服务可关联企业、岗位、面试、offer 和入职状态。
8. 一条通知公告可以面向全部内部用户、指定角色、指定班级或指定人员发布。
9. 每一次重要业务操作应生成操作日志；操作日志与内部用户、操作对象、操作时间和操作结果关联。
10. 前端页面使用的图片、头像、横幅图、课程封面图等静态资源由图片资源数据统一描述，便于资源替换、命名管理和页面维护。

可用文字形式表示局部逻辑数据模型如下。

| 实体 | 主要关联实体 | 关系说明 |
| --- | --- | --- |
| 内部用户 | 角色、操作日志、通知公告 | 内部用户通过角色获得访问权限，并产生操作日志 |
| 角色 | 权限、内部用户 | 角色是权限集合，可分配给多个内部用户 |
| 学员 | 班级、学习进度、作业提交、考试成绩、就业服务记录 | 学员是教学和就业服务的核心管理对象 |
| 班级 | 学员、课程、讲师、内部用户 | 班级承载教学组织、课程安排和学员管理 |
| 课程 | 课程章节、教学资源、作业、考试 | 课程包含教学内容、练习和测评 |
| 课程章节 | 学习进度、教学资源 | 章节用于记录细粒度学习状态 |
| 作业 | 作业提交、课程、班级 | 作业由教师发布，由学员提交并被批改 |
| 考试 | 考试成绩、课程、班级 | 考试用于阶段性考核 |
| 就业服务记录 | 学员、企业、岗位 | 就业记录用于跟踪投递、面试、offer 和入职结果 |
| 图片资源 | 页面、组件、业务对象 | 图片资源用于前端页面展示和资源替换 |
| 统计报表 | 学员、班级、课程、就业服务记录、操作日志 | 报表汇总业务数据并支持筛选分析 |

## 4.2 数据字典

| 数据元素 | 描述 | 组成方式或数据类型 | 长度 | 值/约束 |

| 用户ID | 内部用户的唯一标识 | 字符串或长整型 | 32 | 全局唯一，不允许为空 |
| 用户账号 | 内部用户登录账号 | 字母、数字、下划线 | 6-32 | 唯一；不可与已停用但未注销账号重复 |

| 邮箱 | 用户邮箱地址 | 字符串 | 6-64 | 符合邮箱格式，可为空 |
| 用户状态 | 内部用户可用状态 | 枚举 | 8 | 启用、停用、锁定、待激活 |

| 学员ID | 学员唯一标识 | 字符串或长整型 | 32 | 全局唯一 |

| 学员编号 | 学员在培训机构内的编号 | 字母和数字 | 6-32 | 唯一，可用于检索 |

| 报名日期 | 学员报名日期 | 日期，YYYY-MM-DD | 10 | 不得晚于当前日期 |
| 入班日期 | 学员进入班级日期 | 日期，YYYY-MM-DD | 10 | 不得早于报名日期 |
| 班级ID | 班级唯一标识 | 字符串或长整型 | 32 | 全局唯一 |
| 班级名称 | 班级显示名称 | 字符串 | 2-50 | 如 JavaEE 2026 春季 01 班 |
| 班级状态 | 班级当前教学状态 | 枚举 | 8 | 未开班、开班中、已结课、已归档 |
| 开班日期 | 班级计划或实际开班日期 | 日期，YYYY-MM-DD | 10 | 不允许为空 |
| 结课日期 | 班级计划或实际结课日期 | 日期，YYYY-MM-DD | 10 | 应晚于开班日期 |
| 课程ID | 课程唯一标识 | 字符串或长整型 | 32 | 全局唯一 |
| 课程名称 | 课程展示名称 | 字符串 | 2-80 | 不允许为空 |
| 课程方向 | 课程所属技术方向 | 枚举或字符串 | 2-30 | Java、前端、Python、大数据、人工智能、软件测试等 |

| 章节序号 | 章节在课程中的排列顺序 | 整型 | 4 | 最小值为 1 |
| 资源ID | 教学资源或页面图片资源唯一标识 | 字符串或长整型 | 32 | 全局唯一 |
| 资源名称 | 文件或资源显示名称 | 字符串 | 1-100 | 不允许为空 |
| 资源类型 | 资源文件类型 | 枚举 | 12 | 图片、视频、文档、压缩包、代码、其他 |

| 图片命名 | 前端页面图片资源命名 | 字符串 | 1-80 | 建议格式：`p{页码}-{区块}-{序号}.{扩展名}` |

| 最近学习时间 | 最近一次学习行为发生时间 | 日期时间 | 19 | 格式 YYYY-MM-DD HH:mm:ss |

| 作业标题 | 作业展示标题 | 字符串 | 1-100 | 不允许为空 |
| 作业状态 | 作业发布和处理状态 | 枚举 | 8 | 草稿、已发布、已截止、已归档 |
| 提交ID | 学员作业提交唯一标识 | 字符串或长整型 | 32 | 全局唯一 |

| 考试成绩 | 学员考试得分 | 数值 | 5 | 0-100 |

| 就业记录ID | 就业服务跟进记录唯一标识 | 字符串或长整型 | 32 | 全局唯一 |

| 企业名称 | 招聘企业名称 | 字符串 | 2-80 | 不允许为空 |
| 岗位名称 | 学员投递或录用岗位 | 字符串 | 2-80 | 可为空 |
| 面试时间 | 学员面试安排时间 | 日期时间 | 19 | 不得早于记录创建时间，历史补录除外 |
| 通知ID | 通知公告唯一标识 | 字符串或长整型 | 32 | 全局唯一 |
| 通知标题 | 通知公告标题 | 字符串 | 1-80 | 不允许为空 |
| 通知状态 | 通知发布状态 | 枚举 | 8 | 草稿、已发布、已撤回 |
| 阅读状态 | 用户对通知的阅读状态 | 枚举 | 8 | 未读、已读 |
| 操作日志ID | 操作日志唯一标识 | 字符串或长整型 | 32 | 全局唯一 |
| 操作类型 | 用户操作分类 | 枚举或字符串 | 1-30 | 新增、修改、删除、查询、导入、导出、登录、退出 |
| 操作对象 | 被操作的数据对象 | 字符串 | 1-80 | 如学员、班级、课程、资源、角色 |
| 操作结果 | 操作执行结果 | 枚举 | 8 | 成功、失败 |
| 创建时间 | 数据创建时间 | 日期时间 | 19 | 系统自动生成 |
| 更新时间 | 数据最近更新时间 | 日期时间 | 19 | 更新时自动刷新 |
| 创建人 | 创建数据的用户ID | 字符串或长整型 | 32 | 可追溯到内部用户 |
| 更新人 | 最近更新数据的用户ID | 字符串或长整型 | 32 | 可追溯到内部用户 |
| 删除标记 | 逻辑删除标识 | 布尔 | 1 | false 表示正常，true 表示已删除 |

## 4.3 报表

### 4.3.1 学员综合信息报表

| 报表ID | BM-RPT-1 |
| --- | --- |
| 报表标题 | 学员综合信息报表 |
| 报表目的 | 支持教务、班主任、讲师和就业老师查看学员基础信息、班级归属、学习状态和就业状态，便于进行日常管理和风险跟进 |
| 优先级 | 高 |
| 报表用户 | 系统管理员、教务管理员、班主任、讲师、就业老师 |
| 数据源 | 学员数据、班级数据、课程数据、学习进度数据、就业服务记录 |
| 频率和计划 | 用户按需生成；支持按班级、课程方向、学员状态、报名日期、就业状态筛选 |
| 延时 | 在筛选条件明确且数据量不超过 5000 条时，应在 3 秒内展示第一页结果 |
| 可视化布局 | 表格列表，支持分页、排序、条件筛选和导出 |
| 页头和页尾 | 页头显示报表标题、筛选条件、生成时间；导出文件页脚或末尾显示导出人和数据范围 |
| 报表主体 | 学员编号、学员姓名、手机号、所属班级、课程方向、报名日期、学员状态、学习完成率、最近学习时间、就业状态、负责老师 |
| 报表结束符 | 无 |
| 交互性 | 支持点击学员姓名进入详情页；支持按班级、状态、课程方向筛选；支持导出 Excel |
| 安全访问限制 | 普通讲师和班主任仅能查看授权班级或授权学员；系统管理员可查看全部数据 |

| 可视化布局 | 表格列表 |
| 页头和页尾 | 页头显示查询条件和生成时间 |
| 报表主体 | 操作时间、操作用户、用户角色、操作类型、操作对象、对象ID、请求IP、操作结果、失败原因 |
| 报表结束符 | 无 |
| 交互性 | 支持按用户、时间范围、操作类型、操作结果筛选；支持导出 |
| 安全访问限制 | 仅系统管理员和安全审计角色可访问；日志不允许普通用户删除 |

## 4.4 数据集成、留存和销毁

DI-1：系统应通过统一 API 与后端业务服务进行数据集成，前端不得绕过后端服务直接访问数据库。

DI-2：系统前端展示的数据应以接口返回结果为准；浏览器本地缓存、Pinia/Vuex 状态、SessionStorage 或 LocalStorage 中的数据仅作为临时展示和会话恢复使用，不得作为业务最终数据源。

DI-3：用户登录令牌、刷新令牌、菜单权限、用户基础信息等会话数据应在用户退出登录、令牌过期或账号被停用后立即失效。

DI-4：学员基础信息、班级信息、课程信息、作业、考试、就业服务记录等核心业务数据应至少保留至业务完成后 5 年；如学校或机构管理制度要求更长周期，应按较长周期执行。

DI-5：操作日志、安全日志和登录日志应至少保留 1 年；涉及敏感操作、数据导出、权限变更的日志应至少保留 3 年。

DI-6：教学资源、课程图片、页面图片等静态资源在课程或页面仍处于启用状态时不得删除；资源下架或页面废弃后，应保留不少于 6 个月以支持回溯和恢复。

DI-7：学员手机号、身份证号、邮箱、就业薪资、面试评价等敏感或隐私数据应在存储和传输过程中采取脱敏、访问控制或加密措施。前端列表页默认不得完整展示身份证号等高敏感字段。

DI-8：系统应支持对已删除业务数据进行逻辑删除。普通删除操作只设置删除标记，不应立即物理删除，以便误删恢复和审计追踪。

DI-9：超过留存期限且无继续保留必要的数据，应由具备管理员权限的用户发起销毁流程。销毁前应生成销毁清单，记录数据范围、销毁原因、操作人、审批人和销毁时间。

DI-10：数据销毁应包括业务数据库记录、附件文件、图片资源、缓存副本和导出文件。销毁完成后，系统应保留不可篡改的销毁记录。

DI-11：前端导出的 Excel、CSV、PDF 等文件应只在用户本地保存；系统如需在服务器端生成临时导出文件，应在 24 小时内自动清理。

DI-12：当系统与第三方文件存储、短信、邮件、单点登录或企业微信等外部平台集成时，应记录外部数据同步状态和失败原因，支持管理员在失败后重试。

# 5. 外部接口需求

本章描述“白马程序员内部”前端产品与用户、后端服务、浏览器、硬件环境及外部通信系统之间的接口要求。外部接口需求应确保系统具有一致的交互体验、明确的软件调用边界、可接受的运行环境和可靠的通信机制。

## 5.1 用户界面

UI-1：系统界面应采用统一的内部管理后台风格，页面布局、导航、按钮、表格、表单、弹窗、提示信息、分页器、筛选器和状态标签应保持一致。

UI-2：系统应提供清晰的全局导航结构，包括但不限于首页/工作台、学员管理、班级管理、课程管理、资源管理、作业考试、就业服务、数据统计、通知公告和系统管理等模块。

UI-3：系统应在用户登录后根据其角色和权限动态展示菜单、按钮和数据范围。用户无权限访问的页面不应在导航中显示；直接输入地址访问时应给出无权限提示或跳转到授权页面。

UI-4：系统登录页面应包含账号输入、密码输入、验证码或其他安全校验、登录按钮、错误提示和登录状态反馈。密码输入应以掩码形式显示。

UI-5：列表页面应提供分页、关键字搜索、条件筛选、排序、刷新和空状态展示。列表加载期间应显示加载状态，接口失败时应显示错误提示。

UI-6：表单页面应对必填项、数据格式、长度范围、枚举值和业务规则进行前端校验。校验失败时，应在对应字段附近显示明确提示。

UI-7：新增、编辑、删除、启用、停用、导入、导出等关键操作应有明确的按钮、确认提示和操作结果反馈。删除、停用、撤回、批量导出等风险操作必须二次确认。

UI-8：详情页面应展示对象的基础信息、关联数据、更新时间、创建人和业务状态。对于学员、班级、课程、就业记录等核心对象，应支持从列表页进入详情页。

UI-9：数据统计页面应使用卡片、表格和图表展示关键指标。图表应有标题、单位、图例或提示信息，避免仅依靠颜色表达含义。

UI-10：图片资源相关页面应支持资源预览、资源说明、命名规范提示、上传状态展示和替换结果反馈。

UI-11：系统应适配主流桌面浏览器，最低支持 1366×768 分辨率正常使用；在 1920×1080 分辨率下页面不应出现异常拉伸、重叠或大面积空白。

UI-12：系统应在常用平板宽度下保持可操作性。若部分复杂管理功能不适合移动端，应给出清晰提示而不是出现不可用或错位页面。

UI-13：系统界面应支持键盘基础操作，包括 Tab 切换焦点、Enter 提交主要操作、Esc 关闭弹窗。焦点状态应清晰可见。

UI-14：所有用户可见的成功、失败、警告、确认和空状态提示应使用简洁中文，并能说明用户下一步可执行动作。

UI-15：系统应避免在页面中直接暴露后端错误堆栈、数据库字段名、接口内部异常信息或敏感调试信息。

UI-16：当用户会话过期时，系统应提示用户重新登录，并在安全允许的情况下保留当前页面路径，登录后返回原页面。

UI-17：系统应为长耗时操作提供进度或处理中提示，例如文件上传、批量导入、批量导出和统计报表生成。

UI-18：系统应为导入失败、保存失败、权限不足、网络异常、数据冲突等情况提供可理解的错误提示，并避免用户重复提交造成脏数据。

## 5.2 软件接口

### 5.2.1 后端业务服务接口

SI-1.1：前端应通过 HTTP/HTTPS API 与后端业务服务通信，主要数据格式采用 JSON。

SI-1.2：所有需要登录态的业务接口应在请求头中携带认证令牌，例如 `Authorization: Bearer {token}`。

SI-1.3：接口返回结构应包含状态码、提示信息和数据主体。建议统一格式为：`code`、`message`、`data`、`traceId`。

SI-1.4：前端应调用用户接口获取当前用户信息、角色、菜单权限、按钮权限和数据权限，并据此渲染页面。

SI-1.5：前端应调用学员管理接口完成学员列表查询、详情查询、新增、编辑、状态变更、导入和导出。

SI-1.6：前端应调用班级管理接口完成班级列表、班级详情、学员分班、班级状态维护和班级统计查询。

SI-1.7：前端应调用课程管理接口完成课程列表、课程详情、章节维护、课程发布、课程下架和课程资源绑定。

SI-1.8：前端应调用作业考试接口完成作业发布、提交记录查询、批改结果展示、考试成绩查询和统计分析。

SI-1.9：前端应调用就业服务接口完成就业记录维护、企业信息查询、岗位信息维护、面试跟进和就业统计。

SI-1.10：前端应调用通知公告接口完成通知列表、通知详情、发布、撤回、阅读状态更新和未读数量查询。

SI-1.11：前端应调用操作日志接口完成日志查询、筛选和导出。普通业务用户不应调用日志删除接口。

SI-1.12：前端应对接口异常进行统一处理，包括登录失效、权限不足、资源不存在、参数错误、服务器异常和网络超时。

### 5.2.2 身份认证与权限接口

SI-2.1：系统应提供登录接口，接收账号、密码和验证码等凭证，返回访问令牌、刷新令牌、用户信息和权限摘要。

SI-2.2：系统应提供退出登录接口，使当前令牌失效，并清理前端本地会话数据。

SI-2.3：系统应提供令牌刷新接口，避免用户在正常操作过程中频繁重新登录。

SI-2.4：系统应提供权限校验接口或权限数据接口，用于控制菜单显示、路由访问、按钮可见性和数据范围。

SI-2.5：当用户角色、权限或账号状态发生变化时，系统应在下一次请求、刷新页面或重新登录时及时生效。

### 5.2.3 文件与图片资源接口

SI-3.1：系统应提供文件上传接口，支持课程封面、页面图片、教学资料、作业附件和导入文件上传。

SI-3.2：文件上传接口应限制文件类型、文件大小和文件数量。图片文件建议支持 JPG、PNG、WebP；文档文件可支持 PDF、DOCX、XLSX 等格式。

SI-3.3：系统应提供文件下载或预览接口。用户下载或预览文件前应校验其访问权限。

SI-3.4：图片资源接口应返回资源ID、资源名称、资源URL、资源类型、用途说明、尺寸、创建时间和更新人。

SI-3.5：上传失败时，接口应返回明确失败原因，例如文件过大、格式不支持、网络中断或权限不足。

### 5.2.4 数据导入导出接口

SI-4.1：系统应支持学员、班级、课程资源、就业记录等数据的模板下载、批量导入和导入结果查询。

SI-4.2：导入接口应返回成功条数、失败条数、失败行号和失败原因，前端应允许用户下载失败明细。

SI-4.3：导出接口应支持按当前筛选条件导出，导出内容不得超过用户权限范围。

SI-4.4：大批量导出应采用异步任务方式，前端通过任务状态接口轮询导出进度并在完成后提供下载入口。

### 5.2.5 图表与统计接口

SI-5.1：系统应提供工作台统计接口，返回学员总数、在读人数、班级数量、课程数量、待处理事项、就业进度等汇总指标。

SI-5.2：系统应提供趋势统计接口，支持按日、周、月统计报名人数、学习完成率、作业提交率、考试通过率和就业转化率。

SI-5.3：统计接口应明确统计口径、时间范围和数据更新时间，避免前端页面展示的指标含义不一致。

## 5.3 硬件接口

HI-1：本系统为基于浏览器运行的前端 Web 产品，不直接识别专用硬件接口。

HI-2：用户访问系统所需硬件包括普通办公电脑、笔记本电脑或平板设备。设备应具备可运行主流现代浏览器的 CPU、内存、显示器、键盘和鼠标或触控板。

HI-3：如用户需要上传图片、文档或附件，设备应具备本地文件读取能力，并允许浏览器访问用户主动选择的文件。

HI-4：如系统启用扫码登录、头像拍摄、在线面试记录等扩展功能，浏览器访问摄像头、麦克风等硬件前必须获得用户授权；未授权时应提供替代操作方式。

HI-5：如用户需要打印报表或导出材料，系统应依赖浏览器和操作系统提供的打印接口，不直接控制打印机硬件。

## 5.4 通信接口

CI-1：系统前端与后端服务之间应优先使用 HTTPS 通信，防止账号、令牌、学员信息和业务数据在传输过程中被窃听或篡改。

CI-2：所有接口请求应设置合理超时时间。普通查询接口建议不超过 10 秒；上传、导入、导出和报表生成等长耗时接口可按业务复杂度单独设置。

CI-3：系统应支持跨域访问控制配置。前端部署域名、后端 API 域名和资源域名不一致时，应由后端配置允许的来源、请求头和请求方法。

CI-4：系统应对网络异常、请求超时、服务器不可用和接口返回异常进行统一提示，不应导致页面白屏或功能无响应。

CI-5：系统应使用统一的请求拦截和响应拦截机制处理认证令牌、错误码、重复提交、防抖节流和全局加载状态。

CI-6：对于通知公告、待办提醒、未读消息等实时性要求较高的功能，系统可采用轮询、WebSocket 或 Server-Sent Events 与后端保持通信。若实时通道断开，应自动重连或降级为定时轮询。

CI-7：系统需要发送短信、邮件、企业微信或站内消息时，应由后端消息服务完成实际发送，前端只负责触发请求、展示发送状态和查看发送结果。

CI-8：系统应在通信请求中携带请求追踪标识。当前端展示错误信息时，可向用户显示简短错误编号，便于后端排查问题。

CI-9：批量导入、批量导出、文件上传等接口应支持进度反馈或任务状态查询，避免用户重复点击造成重复任务。

CI-10：系统与第三方服务通信失败时，应记录失败原因并向前端返回可理解的错误信息。前端应提示用户稍后重试或联系管理员。

CI-11：系统应避免在 URL 查询参数中传输密码、令牌、身份证号、手机号等敏感数据。必须传输敏感数据时，应通过请求体并结合 HTTPS 和权限校验完成。

CI-12：系统前端静态资源应支持缓存控制。版本更新后，应确保用户能够加载最新的 JS、CSS 和图片资源，避免旧缓存导致页面异常。
